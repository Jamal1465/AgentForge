# ROADMAP.md

Project: AgentForge  
Status: Implementation roadmap

---

# 1. Roadmap Strategy

AgentForge will be built using verified vertical slices. Each phase must produce runnable behavior and tests before the next phase begins.

---

# 2. Phase Overview

| Phase | Name | Outcome |
|---|---|---|
| 0 | Builder Kit | Complete planning and architecture documents |
| 1 | Foundation | Runnable repository scaffold |
| 2 | Core Contracts | Domain and application contracts |
| 3 | Runtime Core | Plugin registry, routing, workflow state |
| 4 | Memory and Tools | Context packs and governed tool execution |
| 5 | Built-In Agents | Planner, architect, implementation agents |
| 6 | Governance | Security, evaluation, audit, observability |
| 7 | Interfaces | CLI and API |
| 8 | Demo Project | End-to-end generation demonstration |
| 9 | Submission | Capstone assets and final packaging |

---

# 3. MVP Target

The MVP must demonstrate:

- natural-language project intake,
- task planning,
- capability-based agent routing,
- explicit workflow execution,
- artifact generation,
- memory persistence,
- security review,
- evaluation report,
- CLI-driven demo.
